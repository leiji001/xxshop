# XXshopApi — API 接口文档

> 基础路径：`/`（ThinkPHP 多应用模式）  
> 统一响应格式（admin 和 home 模块通用）：
>
> ```json
> {
>   "code": 200,
>   "msg": "ok",
>   "data": {}
> }
> ```
>
> 错误响应示例：
>
> ```json
> {
>   "code": 400,
>   "msg": "错误描述",
>   "data": []
> }
> ```

---

## 一、后台模块（admin）

---

### 1. 管理员登录

| 项目           | 说明                                                               |
| -------------- | ------------------------------------------------------------------ |
| **路径**       | `/admin/admin/login`                                               |
| **HTTP 方法**  | `POST`                                                             |
| **请求参数**   | `username`（string，必填）— 用户名<br>`pass`（string，必填）— 密码 |
| **请求体示例** | `{ "username": "admin", "pass": "123456" }`                        |
| **成功响应**   | `code: 200`，`data` 返回管理员信息（不含 password、salt）          |
| **失败响应**   | `code: 400`，`msg: "管理员不存在"` 或 `"密码错误"`                 |

---

### 2. 管理员列表

| 项目          | 说明                                                             |
| ------------- | ---------------------------------------------------------------- |
| **路径**      | `/admin/admin/list`                                              |
| **HTTP 方法** | `GET`                                                            |
| **请求参数**  | `page`（int，可选，默认 `1`）<br>`limit`（int，可选，默认 `10`） |
| **响应**      | `code: 200`，`data` 为分页后的管理员数组                         |

## 五、注意事项

1. **认证机制**：当前登录接口仅返回用户信息，未实现 token/session 认证，建议后续添加 JWT 或 Session 鉴权。
2. **路由缺失**：`admin/Admin/delete` 方法存在于控制器但未在路由中注册，如需使用请补充。
3. **字段命名不一致**：
   - 数据库中字段为 `picture`，但控制器代码中使用 `image`（如 `GoodsController::add`、`CategoryController::add`），需确认实际使用字段名。
   - admin 模块使用 `is_recoment`（少一个 t），数据库和 home 模块使用 `is_recomented`（多一个 d），需统一。
4. **上传接口**：三个模块都有独立的图片上传接口，分别保存到 `upload/goods`、`upload/category`、`upload/user` 目录。
5. **密码加密**：采用 `md5(md5(password) + salt)` 双重哈希方式。
6. **商品相册**：数据库有 `vueshop_goods_album` 表，但控制器代码中未处理相册的增删改查，需补充。
7. **控制器方法 vs 路由**：
   - `CategoryController::tree()` 在 admin 模块中有定义，但路由中未注册。
   - `CategoryController::update($id)` 接收路由参数 `$id`，但路由 `category/update` 未定义参数模式，实际通过 POST body 传 `id`。
   - `UserController::update($id)` 同理。
